# Code of Conduct

## The short version

Be respectful, assume good faith, and keep disagreements about the code, not
the person. This is a small project maintained in spare time — treat that
context with patience, both ways.

## What that looks like

Examples of behavior that helps:

- Giving and accepting constructive feedback gracefully
- Focusing on what's best for the project and the people using it
- Being direct about disagreements without being unkind about them
- Owning mistakes, apologizing, and learning from them

Examples of behavior that isn't welcome:

- Personal attacks, insults, or derogatory comments
- Harassment, public or private
- Publishing others' private information without consent
- Sexualized language or imagery, or unwelcome sexual attention
- Anything else that would reasonably make someone feel unsafe or unwelcome

## Scope

This applies to project spaces — issues, pull requests, discussions, and any
other official aegis channels — and to anyone representing the project
elsewhere.

## Enforcement

Report unacceptable behavior by opening a private security advisory on
[GitHub](https://github.com/andreipaciurca/aegis/security/advisories/new) or
by reaching the maintainer through their GitHub profile — see
[SECURITY.md](SECURITY.md) for the same channel used for vulnerability
reports. Reports are handled in confidence.

The maintainer may remove comments, commits, code, issues, or other
contributions that don't align with this Code of Conduct, and may temporarily
or permanently ban anyone for behavior they deem inappropriate, threatening,
or harmful.

Trust & verification

aegis is a defensive security tool. It scans files, contains detection strings, can quarantine selected files, can terminate selected processes with confirmation, and can call native firewall tools. Those behaviors are useful, but they can look suspicious to antivirus heuristics.

Verify a release

Official releases should include binaries plus SHA256SUMS and, when available, a detached signature.

gpg --verify SHA256SUMS.asc SHA256SUMS
shasum -a 256 -c SHA256SUMS

Platform signing

  • macOS: releases should be Developer ID signed and notarized when certificates are available.
  • Windows: releases should be Authenticode signed when an OV/EV certificate is available.
  • Linux: verify checksums and detached signatures, or use distro package signatures.

False positives

If an antivirus product flags aegis, submit the binary to the vendor as a false positive with the release URL, SHA-256, source URL (https://github.com/andreipaciurca/aegis) and signed checksum file. Mention that aegis is an open-source defensive scanner and terminal/browser UI.

What aegis does not install

  • No daemon or background service
  • No kernel extension or driver
  • No browser extension
  • No login item
  • No bundled local LLM or model weights

External reputation

Normal scans are local. The optional aegis intel <hash> command can query VirusTotal API v3 for file reputation when you provide an API key. It sends only the MD5, SHA-1 or SHA-256 you type; it does not upload files and it does not run in the background.

Keep your platform security stack enabled. aegis is designed to complement Gatekeeper, XProtect, Defender and EDR tools, not bypass them.

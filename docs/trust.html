Verification guidance

Trust & verification

aegis is a defensive security tool. It scans files, contains detection strings, can quarantine and restore selected files, can terminate selected processes with confirmation, and can call native firewall tools. Those behaviors are useful — but the same strings and behaviors can look suspicious to antivirus heuristics that can't tell intent from a signature match. This page is here so you (or a vendor you submit a false positive to) can verify exactly what you're running.

Verify a release

Official releases include binaries plus SHA256SUMS and, when available, a detached signature.

gpg --verify SHA256SUMS.asc SHA256SUMS
shasum -a 256 -c SHA256SUMS

Platform signing

  • macOS: releases should be Developer ID signed and notarized when certificates are available.
  • Windows: releases should be Authenticode signed when an OV/EV certificate is available.
  • Linux: verify checksums and detached signatures, or use distro package signatures.

False positives

If an antivirus product flags aegis, submit the binary to the vendor as a false positive with the release URL, SHA-256, source URL (github.com/andreipaciurca/aegis) and signed checksum file. Mention that aegis is an open-source defensive scanner and terminal/browser UI — see SECURITY.md for the exact submission checklist.

What aegis does not install

  • No daemon or background service
  • No kernel extension or driver
  • No browser extension
  • No login item
  • No bundled local LLM or model weights

The aegis binary itself never phones home. This website (the page you're reading, not the tool) asks before loading Google Analytics — decline it and nothing loads. That's separate from and unrelated to anything the binary does.

Actions that need your say-so

Every capability that touches something outside the scan itself is explicit, not automatic:

Quarantine → restore

Quarantine strips a file's permissions and logs it; aegis restore reverses it. Restore refuses to run twice or overwrite an existing file.

Process termination

Requires confirmation in the TUI — unlike quarantine, it can't be undone. Refuses to target PID ≤ 1 or aegis itself.

Local browser GUI

Binds to 127.0.0.1 only and checks every request's origin, so a webpage in another tab can't drive it silently.

External reputation

Normal scans are local. The optional aegis intel <hash> command can query VirusTotal API v3 for file reputation when you provide an API key. It sends only the MD5, SHA-1 or SHA-256 you type; it does not upload files and it does not run in the background. The optional aegis clamav <path> command talks only to a local clamd you run yourself.

Keep your platform security stack enabled. aegis is designed to complement Gatekeeper, XProtect, Defender and EDR tools, not bypass them.
